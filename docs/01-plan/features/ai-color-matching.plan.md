# AI 조색 어시스턴트 (AI Color Matching Assistant)

## 1. 개요 (Overview)

### 프로젝트명
**PainterAI** - 자동차 도장 현장용 AI 조색 보조 안드로이드 앱

### 배경 및 동기
자동차 보수도장 현장에서 조색(Color Matching)은 경험과 감각에 크게 의존합니다. 
30년 경력의 베테랑도 같은 컬러에서 비슷한 구조 문제를 반복적으로 겪습니다.
현재 워크플로우의 문제점:
- STD/sample 사진 전달 → 전화 판독 → 조정 지시 → 재확인... 반복으로 시간 손실
- 기록 누락, 판단 편차 발생
- 경험은 쌓이지만 시스템화되지 않음
- 색측기(BYK) 장비는 수백만원대로 고가

### 핵심 가치
"경험이 쌓이면 실력이 되지만, 기록이 쌓이면 시스템이 된다"
- 색측기 = 수치, 사용자 = 경험/판단, AI = 구조 분석/보조 검증
- 핸드폰 사진 + AI 분석으로 29,000원/월 수준의 색측기 보조 시스템 구현
- 매 작업 기록이 축적되어 동일 색상 재작업 시 실패 확률 감소

## 2. 대상 사용자 (Target Users)

### Primary
- 자동차 보수도장 조색 기술자 (5년~30년 경력)
- KCC SUMIX 도료 사용자
- 일일 1~5건 조색 작업 수행

### Secondary
- 도장 공장 관리자 (작업 이력 관리)
- 신입 도장 기술자 (교육용)

### 사용 환경
- 도장 부스/작업장 (분진, 조명 편차)
- 핸드폰으로 작업 중 촬영
- 장갑 착용 상태에서 조작 가능해야 함
- 인터넷 연결 필요 (Claude API)

## 3. 핵심 기능 (Core Features)

### F1. 시편 촬영 및 비교 분석 (MVP - 최우선)
**사용자 플로우:**
1. STD 시편(표준 색상판) 사진 촬영
2. 조색해 만든 시편 사진 촬영
3. STD 시편의 안료 배합(레시피) 입력
4. 조색 시편의 안료 배합(레시피) 입력
5. Claude AI가 사진 + 배합을 비교 분석
6. 어떤 조색제(토너)가 부족/과다한지 사용자에게 알려줌

**AI 분석 항목:**
- 45° (중간각): 메탈릭 구조(입자 크기/밀도/배열), Hue(색상축), a*b* 값 방향
- 105° (측면/암부): Flop(플롭), 입자 영향, 암부 깊이
- 15° (정면): 명도 L*, 스파클, 색상 축 확인
- 톤너별 역할 분석 및 증량/감량 추천
- 실패 리스크 예측

**배합 입력 형식 (KCC SUMIX 기준):**
```
컬러코드: D9B
도료사: KCC SUMIX
배합:
K9001  57.40g
K203   13.94g
K060    6.27g
K702    7.45g
K907    5.37g
K200    3.58g
K918    2.11g
K600    0.96g
K906    2.40g
K805    0.23g
합계:  99.71g
```

### F2. 대화 기록 저장 및 검색
- Claude API와의 대화 내용을 Supabase에 저장
- 차량 모델/컬러코드/연식 기준으로 검색
- 과거 동일 컬러 작업 시 이전 대화 자동 불러오기
- 성공/실패 태그로 필터링

### F3. 작업 아카이브
- 차량코드/연식/부위/레시피/조정톤너 기록
- STD vs Sample 결과 누적 저장
- 사진 + 레시피 + 구조/벨류/색상 + 성공/실패 리스크 기록
- Airtable 스타일의 구조화된 데이터 저장

### F4. 컬러코드 배합 DB (Phase 2)
- KCC SUMIX 배합 데이터 관리
- 컬러코드 검색 시 등록된 배합 자동 로드
- 사용자 커스텀 배합 저장
- 향후 도료사 추가 확장 (노루페인트, 삼화 등)

## 4. 기술 스택 (Tech Stack)

### 클라이언트
- **플랫폼**: Android (Kotlin)
- **최소 SDK**: API 26 (Android 8.0)
- **카메라**: CameraX API
- **UI**: Jetpack Compose
- **로컬 캐시**: Room Database

### 백엔드/데이터
- **데이터 저장**: Supabase (PostgreSQL)
  - 대화 기록 테이블
  - 작업 아카이브 테이블
  - 배합 DB 테이블
  - 사진 스토리지 (Supabase Storage)
- **인증**: Supabase Auth (이메일/비밀번호)

### AI
- **API**: Claude API (Anthropic)
  - 모델: claude-sonnet-4-5-20250514 (비용 효율 + 비전 능력)
  - Vision API로 시편 사진 분석
  - 시스템 프롬프트에 조색 전문 지식 내장
- **예상 비용**: 1회 판독 약 $0.02~0.05 (이미지 2장 + 텍스트)

## 6. 핵심 화면 구성

### S1. 메인 화면 (작업 목록)
- 최근 작업 목록 (차량모델 + 컬러코드 + 날짜)
- 새 작업 시작 버튼
- 검색 (컬러코드, 차량명)

### S2. 새 작업 화면
- 차량 정보 입력 (모델, 연식, 컬러코드, 작업 부위)
- 도료사 선택 (KCC SUMIX / 추후 확장)

### S3. 시편 촬영 및 비교 화면 (핵심)
- STD 시편 촬영 영역
- 조색 시편 촬영 영역
- STD 배합 입력 (토너코드 + g 수)
- 조색 배합 입력 (토너코드 + g 수)
- "AI 분석 요청" 버튼
- AI 분석 결과 표시 영역

### S4. AI 대화 화면
- 채팅 형태의 대화 UI
- 사진 첨부 기능
- 배합표 자동 포맷 전송
- AI 응답에 구조적 분석 결과 표시
- 추가 질문 가능 (예: "105도 암부를 더 자세히 분석해줘")

### S5. 작업 아카이브 화면
- 작업별 상세 기록 조회
- 성공/실패 태그
- 사진 + 배합 + AI 분석 결과 통합 뷰
- 같은 컬러코드 이전 작업 연결

### S6. 배합 DB 화면 (Phase 2)
- 컬러코드 검색
- 등록된 배합 목록
- 배합 직접 입력/수정

## 8. Claude API 시스템 프롬프트 (핵심)

AI에게 주입할 조색 전문가 시스템 프롬프트 핵심 구조:

```
당신은 자동차 보수도장 조색 전문 AI 어시스턴트입니다.

## 역할
- 시편 사진과 배합 데이터를 분석하여 조색 방향을 제안
- STD(표준)와 Sample(조색) 시편을 비교 판독
- 토너별 증량/감량 방향 추천

## 3각도 분석 기준
- 45° (중간각): 메탈릭 구조(입자 크기/밀도/배열), Hue 방향(a*b*)
- 105° (측면): Flop/입자 영향, 암부 깊이, 구조 안정성
- 15° (정면): 명도 L*, 스파클, 페이스 색감

## 판독 순서
1. 105° 먼저 확인 → 구조 문제인지 색상 문제인지 분리
2. 45°로 메탈릭 구조 확인
3. 15°로 최종 색상축 확인

## 분석 출력 형식
- 각도별 차이점 (STD vs Sample)
- 원인 진단 (구조 문제 / 색상 문제 / 둘 다)
- 토너별 조정 방향 (예: K702 미세증량, K600 미세감량)
- 예상 리스크
- 다음 시편 제작 권장사항

## 주의사항
- STD 기준 판독용이며 완벽 일치를 전제하지 않음
- 차체 상태, 도장 조건, 조명에 따라 달라질 수 있음
- 최종 결정은 반드시 사용자(기술자)가 내림
- 배합 g수 조정은 방향만 제안, 구체적 g수는 기술자 판단
```
